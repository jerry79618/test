## 第 2 頁：痛點「當測試，全憑個人經驗」（0:40 – 2:00）

這是我們團隊每天面對的真實現況：**測試，全憑個人經驗。**

【等六張卡浮完，指著卡片講重點，不逐字唸】

第一，測試重複性高——DEV 測一輪、STG 再測一輪，同樣的交易人工重複驗證，測完還要手動做測試報告。

第二，測案因人而異——資深工程師想得到十條案例，新人可能只想得到三條，品質完全看人。

第三，就是最痛的「改 A 壞 B」——回歸測試靠人力記憶，例如說，改了登入查詢功能，壞掉的卻是客戶基本資料變更。

第四 : 涵蓋率沒人說得準，每次上版都像開盲盒
第五: 學習曲線: 測試知識放在個人腦袋裡，新人接手只能口耳相傳；
第六手動測試佔掉開發週期三成以上——需求越急、測試則往往會忽略

這六件事，請大家先記住，等一下我們會一件一件回來解決它。

【往下捲動】

## 第 3 頁：一個需求的旅程（2:00 – 3:20）

痛點聽起來很抽象？我們用一位新人工程師的一天，把它演一次。

【配合站點逐顆亮起，跟著節奏講】

他接到業務單位的需求——第一件事不是寫程式，是**東找西找**：翻 wiki、問前輩，光搞清楚系統要改哪裡就花掉大半天。

好不容易開發完成，接著在 DEV 環境測試——自己建立測試、自己整理測試報告。

上版到 STG——**同樣的事情，全部再來一次**。這位新人已經開始不耐煩了。

上版 UAT，User 驗收**新功能**，一切看起來都沒問題，開開心心上線。

結果一上 PRD——【停一拍，指著紅色警報】——真實用戶**無法進行客戶基本資料變更**。壞掉的，是大家都沒測到的「舊功能」。

【等 punchline 浮出，唸出來】

**測的是新功能，倒下的是舊功能。**這就是「改 A 壞 B」，它不是假設，它就發生在我們身上。

## 第 4 頁：我們的解法（3:20 – 4:40）

所以我們的解法很簡單，一條公式：**Copilot CLI 加上 Cucumber BDD，等於不靠經驗的自動化測試。**

Copilot CLI 這邊，我們做了 Multi-Agent——AI 直接讀懂 JIRA 需求單，自動生成測試案例、自動執行測試。Cucumber 這邊，用的是 Gherkin 語法——中文、英文都能寫，人人看得懂的測案文件。

【指著下方流程圖，由左到右】

整條流水線是這樣：一張 **JIRA 需求單**進來，丟給 **Copilot CLI**，Multi-Agent 開始分析、生成，產出中文的 **Gherkin 測案**——

【滑鼠移到「Gherkin 測案」節點，範例浮出，停 5 秒】

大家看，這就是 AI 產出的真實測案：中文寫的，「當我上傳身分證影像、那麼回應狀態碼應為 200」——PM 看得懂、QA 看得懂、新人第一天就看得懂。而且注意，它有**正向、反向、還有邊界**三種案例。

【滑鼠移開，繼續】

測案接著由 Cucumber-JVM 用 Java 自動執行，最後自動產出 HTML 測試報告。

一行指令、全量覆蓋、秒出報告——而且測案即文件，改 A 壞 B 一鍵回歸立刻現形。

## 第 6 頁：PK 對決——現場 Demo（5:50 – 7:50）

規則很簡單：同一份工作——台幣開戶流程八支 API 的測試案例，從想案例、執行到產出報告。左邊是傳統人工用 Postman；右邊是 Copilot CLI。比的就是**花費時間**。

【按下「開始對決」，配著畫面講】

右邊，我只做一件事：把 JIRA 需求單的網址貼進去。接下來全自動——主 Agent 接單，分派給子 Agent：需求分析 Agent 解析出八支 API、規劃 **24 條測案**，正向、反向、邊界各八條；測案生成 Agent 產出中文 feature 檔；程式碼 Agent 生成 Java 測試程式；Code Review Agent 審查通過，跑 mvn test。

同一時間看左邊——工程師還在想第一條案例怎麼測：打交易、截圖、打開 Excel、一格一格貼結果。

【右側蓋「全部完成」時，指著計時器】

看時間：AI，**十五分鐘**，24 條測案全數通過，Cucumber 的 HTML 報告直接打開——每一條測試、每一個步驟、通過率，清清楚楚。

而左邊呢?【此時左側進入時間快轉】現在畫面開始快轉——同樣的流程他還要再重複十九次。一天過去、兩天過去……

【左側定格，唸結尾大字】

**同一份工作：AI 十五分鐘，產出 24 條含邊界的測案加報告；人工兩個工作天，20 條案例，邊界還想得不全。**

這不是加速的魔術，右邊每一個畫面，就是我們平常真實在用的流程。

## 第 7 頁：我們導入哪些工具（7:50 – 8:40）

支撐這場對決的，是這六個元件。

核心三個：**GitHub Copilot CLI**，終端機裡的 AI 代理，全程自然語言驅動；**Cucumber 加 Gherkin**，讓測案變成人人能讀的活文件；還有我們設計的 **Multi-Agent 協作**——需求分析、測案生成、程式碼、Code Review 四個 Agent 接力，各司其職。

底層由 **Cucumber-JVM 加 Java** 執行測試，自動產出 **HTML Report**，最後整合進 **CI/CD**——每次 commit 自動回歸，問題在上線前就被攔下來。

沒有黑魔法，每一個元件都是今天就能導入的成熟技術，我們做的是把它們串成一條全自動的測試流水線。
